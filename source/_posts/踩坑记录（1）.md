做数据库课设时，我试图修改一个外键的删除规则和更新规则为`CASCADE`。

SQL语句执行成功了，然而刷新后它的规则又都变回了`RESTRICT`。
<!--more-->

我使用的是 MySQL + DBeaver。

建立外键时，选择的删除规则和更新规则是无动作（默认）。

建立后，MySQL把它们设为了`RESTRICT`。

由于一些麻烦的原因，我想把它们改成`CASCADE`，于是出现了上面的情况。

DBeaver给出的SQL语句是把外键删了重建：
```sql
ALTER TABLE dbkeshe.student DROP FOREIGN KEY student_FK;
ALTER TABLE dbkeshe.student ADD CONSTRAINT student_FK FOREIGN KEY (CLASS_NO) REFERENCES dbkeshe.class(CLASS_NO) ON DELETE CASCADE ON UPDATE CASCADE;
```
准确说不是刷新后变回，在当前的外键窗口刷新是不变的，关闭窗口后重新打开就变了。我以为是DBeaver的问题，又在命令行里执行了一遍，还是会变回`RESTRICT`。关键是MySQL还没报错。

网上搜了一下，没有找到类似的情况发生过。

我又尝试把所有的外键约束删了，然后再重建，还是不行。把所有的外键约束和索引删了，然后再重建，还是不行。到最后还是没有找到原因。

我把旧数据库dump出来的sql文件外键约束后面全部加上`ON DELETE CASCADE ON UPDATE CASCADE`，再用sql文件重建一个数据库，这回成功了。然后...我数天后打开它又变回去了？？？并且重建后还是会变回去。

除了手动重建数据库，最后有一个不得已而为之的方法：要想实现级联更新，暂时关闭外键约束

```sql
set foreign_key_checks = 0;
```

然后把要级联更新的所有表都手动更新一遍，再打开约束...
