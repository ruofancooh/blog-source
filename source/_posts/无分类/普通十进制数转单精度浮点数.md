- 在 IEEE 754 标准下

<!--more-->

单精度浮点数，即 32 位二进制浮点数，包含：

一位符号位 + 八位指数位 + 二十三位尾数位

## 例一

以 $0.1_{10}$ 为例：

1. 把十进制转换为二进制
   $0.1 _{10} = 0.0\dot{0}01\dot{1}... _{2}$
2. 用以 $2_{10}$ 为基数的科学计数法表示，并保证【尾数的小数点前是 $1_{2}$】
   $= 1.\dot{1}00\dot{1}..._{2} \times (2^{-4})_{10}$
   可以看到指数是 $-4_{10}$
3. 计算指数位（又叫阶码），用指数加上偏移量 $(2^{e-1}-1)_{10}$

   这里 $e$ 等于【用于表示指数位的位数】 $8_{10}$，所以偏移量为 $127_{10}$

   则阶码为 $-4_{10} + 127_{10} = 123_{10}$

   阶码的二进制为 `0111 1011`，注意 _阶码始终为无符号整数_

4. 取科学计数法的【小数点后二十三位】作为【尾数位】，因为是无限的，所以要舍去一部分

   它的前二十位是：`1001 1001 1001 1001 1001`

   二十一至二十八位是：`1001 1001`

   所以二十一至二十三位，要么取 `100`，要么取 `101`。

   通常是舍入到最接近的，一样接近则偶数（末尾为 `0`）优先，这里 `101` 更接近。

5. 把符号位连同阶码、尾数位缝合起来

   `0 | 0111 1011 | 1001 1001 1001 1001 1001 101`

## 例二

以 $12_{10} = 1.5_{10} \times (2^{3})_{10}$ 为例，和上面的步骤一样：

1. 把十进制转换为二进制

   $12 _{10} = 1100 _{2}$

2. 用以 $2$ 为基数的科学计数法表示，并保证【尾数的小数点前是 $1$】

   $= 1.1_{2} \times (2^3)_{10}$

   可以看到指数是 $3_{10}$

3. 计算指数位（又叫阶码），用指数加上偏移量 $(2^{e-1}-1)_{10}$

   这里 $e$ 等于【用于表示指数位的位数】 $8_{10}$，所以偏移量为 $127_{10}$

   则阶码为 $3_{10} + 127_{10} = 130_{10}$

   阶码的二进制为 `1000 0010`，注意 _阶码始终为无符号整数_

4. 取科学计数法的【小数点后二十三位】作为【尾数位】，因为是有限的，所以后面直接补零

   小数点后二十三位是：`1000 0000 0000 0000 000`

5. 把符号位连同阶码、尾数位缝合起来

   `0 | 1000 0010 | 1000 0000 0000 0000 000`

## 如何验证

- [在线转换](https://baseconvert.com/ieee-754-floating-point)
- [Wikipedia](https://en.wikipedia.org/wiki/IEEE_754) | [维基百科](https://zh.wikipedia.org/wiki/IEEE_754)
- 获取国际标准文件
  IEEE 754 标准是电气与电子工程师协会（Institute of Electrical and Electronics Engineers）制定的关于浮点数表示和运算的标准。最新标准是 [IEEE 754-2019](https://standards.ieee.org/ieee/754/6210/)，对应国际标准 [ISO/IEC 60559:2020](https://www.iso.org/cms/%20render/live/en/sites/isoorg/contents/data/standard/08/09/80985.html)，预计下一次在 2028 年修订。
  http://snti.ru/ 这个网站分享了各种国际标准文件的磁力链接，但是可以免费下载的不全，IEC 的只有 61xxx。据说可以给站长发邮件要 `popov_al@perm.ru`，每个文件一刀乐。在 ISO 官网上下载需要 187 瑞士法郎，合 1000 多人民币。

用 GDB：

```c
#include <stdio.h>

int main()
{
    float num = 1.7;
    printf("");
}
```

在第六行打上断点，VSCode 开调试，在调试控制台里：

```sh
-exec x/4tb &num
```

- `-exec` 执行命令
- `x` 查看内存
- `4` 输出 4 个单元
  - 不写默认为 `1`
- `t` 以二进制形式输出
  - `x` 十六进制
- `b` 以【一个字节】为一个单元
  - `h` 两个字节
  - `w` 四个字节
  - `g` 八个字节
- `&`取变量地址

输出结果：

`0x61fe1c: 10011010 10011001 11011001 00111111`

从右往左排列后：

`00111111 11011001 10011001 10011010`

`0 01111111 10110011001100110011010`

才是`float 1.7`的二进制表示。

还可以直接用

```sh
-exec x/tw &num
```

输出四个字节，结果：

`0x61fe1c: 00111111110110011001100110011010`

## 十进制与 K 进制小数互转

### K 转 10

每一位分别乘上 K 的次方，再相加。

### 2 转 10

**有限小数**

$11.1011_{2} = 3.6875_{10}$

| 项              | 值       |
| --------------- | -------- |
| $1\times2^{1}$  | $2$      |
| $1\times2^{0}$  | $1$      |
| $1\times2^{-1}$ | $0.5$    |
| $0\times2^{-2}$ | $0$      |
| $1\times2^{-3}$ | $0.125$  |
| $1\times2^{-4}$ | $0.0625$ |
| **总和**        | $3.6875$ |

**无限循环小数**

$1.1 \dot{0} 11 \dot{0} ... _{2} = 1.7_{10}$

可以转换成分数计算：

$$
\begin{aligned}
1.1 \dot{0} 11 \dot{0} ... _{2} & = x \\
11.\dot{0} 11 \dot{0} ... _{2} & = 2_{10}x \\
110110. \dot{0} 11 \dot{0} ... _{2} & = 32_{10}x \\
两式相减，110011_{2} & = 30_{10}x \\
x & = \frac{110011_{2}}{30_{10}} = \frac{51_{10}}{30_{10}} = 1.7_{10}
\end{aligned}
$$

## 10 转 K

整数部分仍然除 K 取余，小数部分乘 K 取整，然后相加。

### 10 转 2

$0.7 _{10} = 0.1 \dot{0} 11 \dot{0} ... _{2}$

| 小数部分乘 2         | 取整数部分 |
| -------------------- | ---------- |
| $0.7 \times 2 = 1.4$ | $1$        |
| $0.4 \times 2 = 0.8$ | $0$        |
| $0.8 \times 2 = 1.6$ | $1$        |
| $0.6 \times 2 = 1.2$ | $1$        |
| $0.2 \times 2 = 0.4$ | $0$        |
| ...                  | ...        |

### 10 转 16

$0.7 _{10} = 0.\mathrm{B} \dot{3} ... _{16}$

| 小数部分乘 16          | 取整数部分   |
| ---------------------- | ------------ |
| $0.7 \times 16 = 11.2$ | $\mathrm{B}$ |
| $0.2 \times 16 = 3.2$  | $3$          |
| ...                    | ...          |
