## node模块安装路径问题

npm安装模块有两种安装方式：本地安装和全局安装。

本地安装：`npm install <pageName>`，安装在当前文件夹下的node_modules目录。

全局安装：`npm install <pageName> -g`，默认安装在`C:\Users\用户名\AppData\Roaming\npm`下。这好吗，这不好。

可使用`npm config get prefix`查看当前全局安装路径。

使用`npm config set prefix "路径"`更改全局安装路径。

设置完后不要忘了改环境变量。

## 把博客放在子文件夹/blog下：

- github新建一个名为blog的仓库，branch为gh-pages

- 更改hexo的_config.yml文件以下项：

```yml
url: https://ruofancooh.github.io/blog
root: /blog/
deploy:
  type: git
  repo: https://github.com/ruofancooh/blog
  branch: gh-pages
```

## hexo更换next主题

在博客目录下运行：

```sh
git clone https://github.com/theme-next/hexo-theme-next themes/next
```

更改站点配置文件中`theme`字段的值为`next`

## next主题设置背景图片

把主题配置文件中的`custom_file_path:`下的`style`取消注释

在`next/source/`下新建`_data`目录，目录里新建`styles.styl`文件。

在`styles.styl`文件中设置：

```css
body {
    /*注意用相对路径，然后在相应的路径下放图片*/
    background-image: url(../images/background.png); 
    background-repeat: no-repeat;    /*不重复*/
    background-attachment: fixed;    /*不滚动*/
    background-size: cover;    /*图片尺寸不拉伸*/
}
```

## 在博客中插入图片

hexo配置文件中修改：

```yml
post_asset_folder: true
```

`_posts`文件夹下新建与md文档同名的文件夹，放入图片。在md文档中用

`{% asset_img filename.png %}`引入图片。

## 更换markdown渲染器

为使博客支持latex，卸载自带渲染器并更换渲染器：

```sh
npm un hexo-renderer-marked
npm i hexo-renderer-pandoc
```

需要额外[安装pandoc](https://github.com/jgm/pandoc/releases/latest)

更改next配置文件：

```yml
math:
  every_page: true

  mathjax:
    enable: true
```
